# Security Policy

If you discover a security vulnerability in this project, please open a private issue or email the repository owner.

We appreciate responsible disclosure and will credit you if desired.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

1. **Do not** open a public issue
2. Send a detailed report to the repo owner via GitHub private vulnerability reporting
3. Include steps to reproduce and potential impact
